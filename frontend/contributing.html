Contributing Code

We follow the standard GitHub fork-and-pull-request workflow. Here's exactly how to contribute code to RoyaleForge.

1
Fork the Repository
Click the "Fork" button on the GitHub repository page to create your own copy.
2
Clone Your Fork
Clone your forked repository locally.
Terminal 
git clone https://github.com/YOUR-USERNAME/royaleforge.git
cd royaleforge
3
Create a Branch
Create a new branch with a descriptive name following the convention.
Branch Naming 
# Feature branches
git checkout -b feature/add-deck-export

# Bug fix branches
git checkout -b fix/card-image-loading

# Documentation branches
git checkout -b docs/update-api-reference
4
Make Your Changes
Write your code following our standards. Test thoroughly before committing.
5
Commit with Clear Messages
Use conventional commits for clear history.
Git 
git add .
git commit -m "feat: add deck export to JSON"
git push origin feature/add-deck-export
6
Open a Pull Request
Go to GitHub and open a PR. Fill out the template with a clear description of what changed and why.

Coding Standards

Python

  • Follow PEP 8 style guide
  • Use type hints for function signatures
  • Write docstrings for all public functions and classes
  • Keep functions focused — one responsibility per function
  • Use meaningful variable names

JavaScript

  • Use ES6+ syntax (const, let, arrow functions)
  • Prefer async/await over promise chains
  • Comment complex logic, not obvious code
  • Keep functions small and pure when possible

CSS

  • Use CSS variables from the design system
  • Follow BEM-like naming for new components
  • Mobile-first responsive design
  • Avoid !important unless absolutely necessary

Commit Message Format

We use Conventional Commits for consistent commit history.

Format 
<type>(<scope>): <description>

# Examples:
feat(analyzer): add deck export to JSON
fix(api): correct rarity filter not working
docs(readme): update installation instructions
style(navbar): improve mobile responsiveness
refactor(ml): optimize model loading
test(decks): add validation tests
Commit Types
feat for features, fix for bug fixes, docs for documentation, style for formatting, refactor for code changes, test for tests.

We are committed to providing a welcoming and inclusive environment. All contributors are expected to:

  • Welcome newcomers and help them get started
  • Accept constructive feedback gracefully
  • Focus on what's best for the community
Zero Tolerance
Harassment, discrimination, or toxic behavior will result in immediate removal from the project.
