Found a bug?

Help us improve RoyaleForge by reporting bugs with clear details. Quality reports lead to faster fixes.

Before Reporting

Save time by checking these first — your issue might already be known.

  • Search existing issues on GitHub
  • Make sure you're on the latest version
  • Try to reproduce the bug consistently
  • Check the browser console for errors (F12)

How to Write a Good Bug Report

Good bug reports have these qualities. Follow this structure for fastest results.

1
Clear Title
Summarize the bug in one line. Bad: "It doesn't work". Good: "Deck analyzer returns 500 error when using champion cards".
2
Describe the Bug
Explain what's happening vs what should happen. Be specific.
3
Steps to Reproduce
Number each step clearly. Anyone should be able to follow them and see the bug.
4
Include Screenshots
A picture is worth a thousand words. Screenshots and screen recordings dramatically speed up debugging.
5
Add Environment Info
Share OS, browser, Python version, and any other relevant details.

Bug Report Template

Copy this template when opening a new issue on GitHub.

Markdown Template 
## Bug Description
A clear and concise description of what the bug is.

## Steps to Reproduce
1. Navigate to '...'
2. Click on '...'
3. Enter '...'
4. See error

## Expected Behavior
What you expected to happen.

## Actual Behavior
What actually happened instead.

## Screenshots
If applicable, add screenshots to help explain.

## Environment
- OS: [e.g., Windows 11, macOS Sonoma]
- Browser: [e.g., Chrome 120, Firefox 121]
- Python Version: [e.g., 3.10.5]
- RoyaleForge Version: [e.g., 1.0.0]

## Additional Context
Any other context about the problem.
Pro Tip
Include browser console output by pressing F12 → Console tab. Copy any error messages and paste them into your report.
